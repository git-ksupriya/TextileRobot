Mathematical Logic

This section describes the mathematical models used for robotic arm motion planning and colour detection.

Inverse Kinematics

The robot arm is modeled as a two-link planar manipulator with a rotating base joint.

Base rotation:

θbase = atan2(y,x)

Planar distance from base:

r = √(x² + y²)

Using the cosine rule for the elbow joint:

cos(θ₂) = (r² + z² − L₁² − L₂²) / (2L₁L₂)

θ₂ = acos(cos(θ₂))

Shoulder angle:

θ₁ = atan2(z,r) − atan2(L₂ sinθ₂ , L₁ + L₂ cosθ₂)

Servo Angle Limits

Servos can only rotate between 0° and 180°. Therefore computed angles are clamped within this range before being sent to the hardware.

Servo PWM Control

Servo motors are controlled using PWM signals with a frequency of 50 Hz.

Pulse width corresponds to the servo angle:

Duty = min_duty + (angle / 180) × (max_duty − min_duty)

Linear Trajectory Planning

The robot moves between two points using linear interpolation.

x(t) = x₀ + t(xf − x₀)

y(t) = y₀ + t(yf − y₀)

z(t) = z₀ + t(zf − z₀)

where t varies from 0 to 1.

Colour Sensor Principle

The colour sensor contains photodiodes with red, green and blue filters. The sensor outputs a frequency proportional to the light intensity detected by each filter.

By comparing the frequencies of red, green and blue components, the dominant colour of the object can be determined.

Calibration Considerations

  • Servo limits were clamped between 0° and 180° to prevent hardware damage.
  • Colour readings require a short delay to stabilize sensor output.
  • Linear interpolation was used to produce smooth robotic motion.
